# Technologies Used in ShopRite E-Commerce

This document explains the main technologies and languages used in the ShopRite Flutter e-commerce project, and where each one appears in the codebase.

## Language Breakdown

The project is mostly written in Dart because it is a Flutter application. The other languages are used by Flutter's platform-specific folders for Android, iOS, web, Windows, Linux, and macOS builds.

| Technology | Approx. Share | Where It Is Used | Purpose |
| --- | ---: | --- | --- |
| Dart | 83.1% | `lib/`, `test/` | Main app UI, screens, providers, repositories, models, services, Firebase logic, Razorpay payment flow, recommendation engine, cart, wishlist, orders, and tests. |
| C++ | 8.4% | `windows/runner/`, `windows/flutter/`, `linux/runner/`, `linux/flutter/` | Native desktop runner code generated by Flutter for Windows and Linux builds. It starts the Flutter engine and connects the app to desktop windows. |
| CMake | 6.6% | `windows/CMakeLists.txt`, `linux/CMakeLists.txt`, platform CMake files | Build configuration for Windows and Linux desktop targets. It tells the native compiler how to build the desktop runner. |
| Swift | 0.9% | `ios/Runner/`, `macos/Runner/`, `macos/Flutter/` | Native iOS and macOS runner code. It launches the Flutter app and registers platform plugins. |
| HTML | 0.5% | `web/index.html` | Web entry point for the Flutter web build. It loads the generated Flutter JavaScript bundle in the browser. |
| C | 0.5% | Generated/native platform support files | Low-level native support used by Flutter tooling or platform build files. It is not part of the main business logic. |

## Main App Technologies

### Flutter

Flutter is the core UI framework of the project. It is used to build the full shopping app experience from a single Dart codebase.

Used in:

- `lib/app.dart`
- `lib/main.dart`
- `lib/screens/`
- `lib/widgets/`
- `lib/core/theme/`

Flutter powers the home screen, product listing, product detail page, cart, wishlist, checkout, order tracking, profile screen, admin dashboard, and reusable widgets.

### Dart

Dart is the main programming language of the application. All app logic is written in Dart.

Used for:

- Models in `lib/models/`
- Providers in `lib/providers/` and `lib/state/`
- Firebase services in `lib/services/firebase/`
- Local cache in `lib/services/local/cache_service.dart`
- Product, cart, order, payment, and auth repositories in `lib/repositories/`
- Recommendation and trending logic in `lib/logic/`
- Tests in `test/`

### Firebase

Firebase is used for backend services.

Used in:

- `lib/firebase_options.dart`
- `lib/services/firebase/auth_service.dart`
- `lib/services/firebase/firestore_service.dart`
- `lib/services/firebase/storage_service.dart`
- `firebase.json`
- `android/app/google-services.json`

Firebase Auth handles login and signup. Cloud Firestore stores user and product data. Firebase Storage support is prepared for storing uploaded files or product images.

### Provider and Riverpod

State management is handled with Provider and Riverpod.

Used in:

- `lib/providers/auth_provider.dart`
- `lib/providers/cart_provider.dart`
- `lib/providers/product_provider.dart`
- `lib/providers/theme_provider.dart`
- `lib/providers/wishlist_provider.dart`
- `lib/state/checkout_provider.dart`
- `lib/state/order_provider.dart`
- `lib/state/payment_provider.dart`

These files manage login state, product loading, cart changes, wishlist updates, checkout state, orders, payment history, and app theme changes.

### Razorpay

Razorpay is used for the payment checkout flow.

Used in:

- `lib/config/razorpay_config.dart`
- `lib/services/payment_service.dart`
- `lib/repositories/payment_repository.dart`
- `lib/state/checkout_provider.dart`
- `lib/state/order_provider.dart`

The app uses Razorpay Test Mode for Pay Now payments. Pay Later orders are also supported with pending payment status.

### Cloud Firestore

Cloud Firestore stores structured app data such as users and products.

Used in:

- `lib/services/firebase/firestore_service.dart`
- `lib/repositories/auth_repository.dart`
- `lib/repositories/product_repository.dart`

It supports fetching product data, creating user records, and reading user-related information.

### Shared Preferences

Shared Preferences is used for small local data storage.

Used in:

- `lib/services/local/cache_service.dart`

It stores lightweight cached data such as recently viewed products, cached product lists, or user-side local app data.

### Cached Network Image

Cached Network Image is used to load and cache product images from URLs.

Used in:

- Product cards
- Product listing UI
- Product detail UI
- Home screen product sections

This improves image loading performance and gives a smoother shopping experience.

### Connectivity Plus

Connectivity Plus is used to detect network status.

Used in:

- `lib/core/utils/network_checker.dart`

It helps the app show offline handling and cached product data when the user has no internet connection.

### FL Chart

FL Chart is used for analytics charts.

Used in:

- `lib/domain/order_analytics.dart`
- Payment or order history analytics screens

It helps display monthly spending and order analytics visually.

### Intl

Intl is used for formatting values such as currency, dates, and numbers.

Used across UI and business logic where readable formatting is needed for prices, dates, or order information.

## Platform Folders

Flutter creates platform folders so the same Dart app can run on different devices.

| Folder | Platform | Main Technology |
| --- | --- | --- |
| `android/` | Android | Kotlin, Gradle, Android Manifest |
| `ios/` | iPhone/iPad | Swift, Xcode project files |
| `web/` | Browser | HTML and generated Flutter web output |
| `windows/` | Windows desktop | C++, CMake |
| `linux/` | Linux desktop | C++, CMake |
| `macos/` | macOS desktop | Swift, Xcode project files |

Most manual development happens in `lib/`. The platform folders mainly contain generated runner code and configuration needed to build the Flutter app for each operating system.

## Summary

ShopRite is a Flutter and Dart e-commerce app. Dart contains the main application logic and user interface. Firebase provides authentication and database services. Razorpay handles test payments. Provider and Riverpod manage app state. The smaller amounts of C++, CMake, Swift, HTML, and C come from Flutter's platform support for desktop, iOS, macOS, and web builds.
